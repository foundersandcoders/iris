01 What is Iris?

Iris is a tool that turns your learner spreadsheet exports into the specific file format the ESFA (Education and Skills Funding Agency) requires for apprenticeship data submission.

The ESFA needs data submitted in a format called ILR (Individualised Learner Record), specifically as an XML file (a structured text file with a .XML extension). Creating these files by hand would be painful. Iris does it for you: you give it a CSV file (a spreadsheet export, the kind you download from Airtable), and it produces the correct XML file.

Iris lives in the terminal, a text-only window you can open with Spotlight by searching for Terminal. This might feel unfamiliar at first, but Iris is fully menu-driven. You never need to type commands or remember syntax; you just use the arrow keys to pick options and press Enter.

✓ You can’t break anything

Iris only reads your CSV and writes a new XML file to a separate folder. It never modifies, moves, or deletes your original spreadsheet. Your Airtable data is completely untouched.

02 Starting Iris

  1. 1

    Open the Terminal application. You can find it by pressing ⌘ Cmd + Space to open Spotlight and typing Terminal, then pressing Return.

  2. 2

04 The main menu

Every time you run iris you land here. All eight options are listed below. The ones you will use most often are 1, 2, and 3.

1
Convert CSV to ILR XML
The main job. Turns your Airtable export into the XML file you submit to the ESFA.
2
Validate XML Submission
Checks an existing XML file (or CSV) for errors without converting anything. Useful for spotting problems before submission.
3
Cross-Submission Check
Compares your current XML against a previous one to catch unexpected changes, like a big drop in learner numbers.
4
Mapping Builder
For advanced use only. Defines how your CSV column headings map to ILR fields. The default mapping already works with the standard FAC Airtable export.
5
Browse Submission History
A log of every XML file Iris has created. You can re-validate or run a cross-check on any past submission from here.
6
Settings & Configuration
Set your provider name and UKPRN number. These go into every XML file Iris produces.
7
About Iris
Shows the software version, Bun runtime version, and platform details.
8
Quit
Exit Iris. You can also press q or Ctrl+C from anywhere.

05 First-time setup

Iris works out of the box without any configuration. However, before you create your first submission for real, you should enter your provider details so they appear correctly in every XML file.

⚠ Set your UKPRN before your first conversion

The UKPRN (UK Provider Reference Number) is an 8-digit number assigned to your organisation by UKRLP. Iris ships with a placeholder value of 10000000. If you convert without changing it, your XML file will contain this placeholder and will be rejected by the ESFA.

How to set your provider details

  1. 1

    From the main menu, press 6 or arrow to Settings & Configuration and press Enter.

  2. 2

    Arrow down to UKPRN and press Enter to edit it. Type your 8-digit UKPRN, then press Enter to confirm.

  3. 3

    Arrow down to Name and press Enter to edit it. Type your provider name, then press Enter.

  4. 4

    Press s to save. You will see “✓ Saved!” at the bottom if it worked.

  5. 5

    Press Esc to return to the main menu.

✓ Everything else is ready to go

Iris ships with the correct rulebook (called a schema) for the 2025–26 collection year and a mapping that already understands the standard FAC Airtable export format. You do not need to configure either of these.

06 Task: Convert a CSV to an XML submission

This is the main thing Iris does. You give it your Airtable export (a .csv file) and it produces an XML file ready to submit.

Before you begin

Export your learner data from Airtable as a CSV file and save it somewhere you can find it; Downloads or Desktop are good choices. The file must have .csv as its extension.

Steps

  1. 1

    From the main menu, press 1 or select Convert CSV to ILR XML and press Enter.

  2. 2

    A file browser appears. At the top you will see where you currently are (for example ~/Downloads). The list shows folders (marked with >) and .csv files.

    Select CSV File
    ~/Downloads > Airtable/ > Old exports/ learners-2526.csv previous-draft.csv
    [↑↓] Nav    [ENTER] Select    [BACKSPACE] Up Dir    [ESC] Back

    Use ↑ and ↓ to move. Press Enter on a folder to open it. Press Backspace to go up one folder. Press Enter on a .csv file to select it.

  3. 3

    Once you select a file, Iris runs automatically. You will see four steps tick off one by one:

    STEP 1
    Parse CSV
    →
    STEP 2
    Validate Data
    →
    STEP 3
    Generate XML
    →
    STEP 4
    Save Output

    The whole process usually takes a few seconds.

  4. 4

    If everything succeeds, you will see a success screen showing the filename of your new XML file and how many learners it contains. Your file is now in ~/Documents/Iris/Submissions/ (that is, your Documents folder → Iris folder → Submissions folder).

    ✓ Where to find your XML file

    Open Finder, go to Documents → Iris → Submissions. The filename will look like ILR-10000000-2526-20260630-143005-01.XML, where the long number is today’s date and time. This is the file you upload to the ESFA portal.

What happens if errors are found

During Step 2 (Validate Data), Iris checks every row for problems. If it finds any errors, it stops immediately: steps 3 and 4 are skipped and no XML file is created. You will be taken to the Validation Issues screen (see Section 9).

⚠ Errors block conversion; warnings do not

Errors (marked ✗) must be fixed in your spreadsheet before Iris can produce an XML file. Export a corrected CSV from Airtable and try again.

Warnings (marked (!)) are noted but do not stop conversion. You can choose to investigate them or proceed.

07 Task: Validate an existing file

Use this to check a file for problems without converting anything. You can validate either a .csv spreadsheet export or an existing .xml submission.

  1. 1

    Press 2 from the main menu, or select Validate XML Submission.

  2. 2

    The file picker appears, showing both .csv and .xml files. Navigate to and select the file you want to check.

  3. 3

    Iris runs four steps: Load File → Parse File → Run Validation → Generate Report.

  4. 4

    No issues found: a success screen confirms the file is clean.

    Issues found: the Validation Issues screen opens (see below).

Use ↑ ↓ to scroll through the issue list. Use ← → to switch between the Errors, Warnings, and All tabs. Each issue shows the field name, the row number, and a short description of what is wrong.

To fix errors, correct the relevant rows in your Airtable base, export a fresh CSV, and run the conversion again.

08 Task: Cross-submission check

This compares your latest XML file against a previous one to spot anything unexpected before you submit. Think of it as a “does this look right compared to last time?” sanity check.

  1. 1

    Press 3 from the main menu, or select Cross-Submission Check.

  2. 2

    A file picker appears with the title Select Current XML Submission. Navigate to and select your new (most recent) XML file.

  3. 3

    A second file picker appears with the title Select Previous XML Submission. Navigate to and select your previous XML file to compare against.

  4. 4

    Iris runs the check and shows you a results screen comparing the two files.

What the check looks for

Finding What it means Severity
Learner count changed by more than 30% Big jump or drop in the number of learners, worth double-checking before submitting Warning
Duplicate learner references found The same learner appears more than once in the current file Error
Collection year changed The academic year in the file has changed, expected when starting a new year Info
First submission (nothing to compare) No previous file exists; this is your first submission Info

09 Understanding validation messages

When Iris finds a problem in your CSV, it shows an error message in plain English. Here are the most common ones and how to fix them in Airtable:

Message (paraphrased) What it means What to fix
Required column “X” is missing The expected column heading is not present in the CSV at all Make sure the column exists in Airtable and is included in the export
Required field “X” is missing, row N A cell that must be filled in is blank for that learner Open the Airtable record and fill in the missing value, then re-export
Field “X” must be an integer (whole number), row N The value contains letters, symbols, or a decimal point where only whole numbers are allowed Correct the value (e.g. 123.0 should be 123)
Field “X” must be a date (YYYY-MM-DD), row N The date is in the wrong format Change the date format in Airtable to year-month-day (e.g. 1999-04-15)
Value “X” is not in the allowed list, row N The value must be one of a fixed set of codes, and this one isn’t recognised Check the ESFA ILR specification for valid codes, then correct the Airtable record
Field “X” must be at least N characters The value is too short Check the value in Airtable; a reference number or code may be truncated
Field “X” must be no more than N characters The value is too long Shorten the value; free-text fields sometimes exceed the ILR character limit
Value “X” does not match the required pattern The value doesn’t fit the expected format (e.g. a postcode is malformed) Correct the format in Airtable (e.g. UK postcodes should be like EC1A 1BB)
✓ Row numbers are 1-indexed

When Iris says “row 4”, it means the 4th data row, not counting the heading row. So “row 1” is the first learner, “row 4” is the fourth learner, and so on.

10 Browsing submission history

Press 5 from the main menu to see a list of every XML file Iris has ever created. Each entry shows the filename, the date it was created, and the number of learners it contains.

KeyAction
↑ ↓Move through the list
Enter or vValidate the selected submission
cRun a cross-submission check starting from the selected file
xDelete a broken entry (press x twice to confirm)
Esc / q / BackspaceGo back to the main menu
ⓘ What “broken” means

If an entry is marked as broken, it means the XML file it references has been moved or deleted from ~/Documents/Iris/Submissions/. Iris keeps the history record but flags it so you know the file is gone. You can press x twice to remove the broken entry from the list.

12 Where your files live

Iris keeps your data in two places. The first is where your real output files go; the second is where Iris stores its own settings.

Location What is stored there Do you need to open it?
~/Documents/Iris/Submissions/ Your finished XML files, the ones you upload to the ESFA Yes, regularly. This is where you find your output.
~/Documents/Iris/Schemas/ Optional: custom rulebook (XSD) files you have added yourself Only if you use a custom schema.
~/.iris/ Iris’s own settings, field mappings, and submission history, a hidden folder Rarely. Iris manages this itself. Do not delete it.
✓ Your data survives every update

When you update Iris to a new version, all of your settings, submission history, and XML output files are untouched. Updating Iris only replaces the program itself.

Finding your Documents folder in Finder

Open Finder, click Documents in the left sidebar, then open the Iris folder, then Submissions. Your XML files will be there, named with the date and time they were created.

ⓘ If you do not have a Documents folder

On some macOS setups the Documents folder is managed by iCloud Drive and may not appear in the usual place. If Iris cannot find ~/Documents/Iris/ it will fall back to ~/Iris/ (a folder called Iris directly in your home folder). Either location works identically.

13 Glossary

ILR
Individualised Learner Record
The national standard for reporting apprenticeship and further-education learner data to the ESFA.
ESFA
Education and Skills Funding Agency
The UK government body that funds apprenticeships and further education. You submit ILR data to them.
CSV
Comma-Separated Values
A simple spreadsheet file format. Airtable’s “Export as CSV” produces one of these.
XML
Extensible Markup Language
A structured text file format. The ESFA requires ILR submissions to be in this format. Iris creates it for you.
UKPRN
UK Provider Reference Number
An 8-digit number that identifies your organisation, assigned by the UKRLP register. Required on every submission.
Schema
XSD, XML Schema Definition
The rulebook file that defines what a valid ILR XML file must look like. Iris ships with the 2025–26 version built in.
Mapping
(CSV-to-ILR field mapping)
The configuration that tells Iris which column in your CSV corresponds to which field in the ILR. The default works with the FAC Airtable export.
Collection year
(Academic year code)
The academic year a submission covers, written as a 4-digit code. For 2025–26 it is 2526.
Terminal
(Command-line interface)
A text-based window on your Mac where you type short commands. Iris runs here, but you only ever need to type one word: iris.
