Distributing & Installing Iris via GitHub Releases

foundersandcoders/iris — standalone binary, macOS (Apple Silicon)

standalone binary no Bun required on target ~62 MB · darwin-arm64

Iris compiles into a self-contained executable via Bun’s --compile mode. The Bun runtime, all npm dependencies, and the ESFA schema are embedded inside a single binary. The target machine needs no Bun, no Node, and no clone of the repository.

01 Distribute as a GitHub Release

Cutting a release involves building the binary for each target platform, then publishing it as a GitHub Release asset. You can do this manually (the proven path) or automate it so every version tag triggers a build.

A — Manual release (the proven path)

This mirrors how v5.0.0 was cut. You run the build locally, then create the GitHub Release with the gh CLI.

  1. 1

    Bump the version (optional but recommended).

    version:next reads your conventional-commit history via svu, picks the correct semver bump, and rewrites package.json, src-tauri/Cargo.toml, src-tauri/tauri.conf.json, and the README.md header. Commit and tag before building.

    shell 
    bun run version:next            # auto-detect bump from commits (needs svu)
# or pin a specific version:
bun run version:set 5.1.0

git add package.json src-tauri/Cargo.toml src-tauri/tauri.conf.json README.md
git commit -m "chore: bump version to 5.1.0"
git tag v5.1.0
git push origin main --tags
  2. 2

    Build the macOS binary.

    The build script (scripts/build-tui.ts) compiles src/cli.ts into a standalone Mach-O executable at dist/iris-darwin-arm64. On macOS it automatically applies an ad-hoc code signature (codesign --sign -) — a non-fatal step that reduces some Gatekeeper friction.

    shell 
    bun run build:tui:mac
# Output: dist/iris-darwin-arm64  (~62 MB, Mach-O arm64)
  3. 3

    Verify the binary before releasing.

    shell 
    dist/iris-darwin-arm64 help
# Expected: === Iris Help === usage printed
  4. 4

    Create the GitHub Release and attach the binary.

    --generate-notes writes changelog entries from commits since the last tag. The binary is attached directly as a release asset.

    shell 
    gh release create v5.1.0 dist/iris-darwin-arm64 \
  --repo foundersandcoders/iris \
  --title "Iris v5.1.0" \
  --generate-notes
    ⓘ Asset naming

    The asset will appear in the release as iris-darwin-arm64. Installers download it by this name (see Section 2). If you later build an Intel binary, attach it as iris-darwin-x64 in the same command.

B — Automate with GitHub Actions

Push a version tag (git push --tags) and GitHub Actions builds and publishes the release automatically. Create this file in your repo:

⚠ Existing .github/release.yml is inert

The repo contains .github/release.yml which is a commented-out Windows/ PowerShell template with no working steps. It does not live in .github/workflows/ so GitHub never runs it. The workflow below is a separate, functional file that fills this gap.

.github/workflows/release.yml — create this file 
name: Build and Release Iris TUI

on:
  push:
    tags:
      - 'v*'   # fires on: git tag v5.1.0 && git push --tags

jobs:
  build-macos:
    name: Build macOS binary
    runs-on: macos-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Set up Bun
        uses: oven-sh/setup-bun@v2
        with:
          bun-version: latest

      - name: Install dependencies
        run: bun install --frozen-lockfile

      - name: Build TUI binary (Apple Silicon)
        run: bun run build:tui:mac
        # Outputs: dist/iris-darwin-arm64

      - name: Verify binary
        run: dist/iris-darwin-arm64 help

      - name: Create GitHub Release and upload asset
        uses: softprops/action-gh-release@v2
        with:
          files: dist/iris-darwin-arm64
          generate_release_notes: true
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
✓ Release flow with Actions

Once the workflow is in place, the full release cycle is:

bun run version:next            # bump version
git add -p && git commit -m "chore: bump version"
git tag v$(node -p "require('./package.json').version")
git push origin main --tags     # ← triggers the workflow

The Actions run builds the binary, creates the GitHub Release, and attaches iris-darwin-arm64 as an asset. No further manual steps.

02 Install on a New Machine

The compiled binary is fully standalone. The target machine needs no Bun, no Node, no npm — just macOS on Apple Silicon.

ⓘ Intel Mac?

The current release ships a single iris-darwin-arm64 (Apple Silicon) asset. For Intel Macs you need an iris-darwin-x64 build (bun run scripts/build-tui.ts --target darwin-x64) attached to the same release. Substitute iris-darwin-x64 for the binary name below.

  1. 1

    Download the binary from the latest GitHub Release. Either download from the web UI, or use the gh CLI:

    shell 
    # via gh CLI (recommended)
gh release download --repo foundersandcoders/iris \
  --pattern 'iris-darwin-arm64' \
  --dir ~/Downloads

# or: download manually from
# https://github.com/foundersandcoders/iris/releases/latest
  2. 2

    Make the binary executable.

    shell 
    chmod +x ~/Downloads/iris-darwin-arm64
  3. 3

    Remove the macOS quarantine flag.

    macOS automatically quarantines files downloaded from the internet. The xattr command strips the quarantine attribute so Gatekeeper will not block the binary on first run.

    shell 
    xattr -d com.apple.quarantine ~/Downloads/iris-darwin-arm64
# If it prints "No such xattr" — that is fine. The attribute was not present.
    ⚠ Gatekeeper may still block the binary

    Iris is ad-hoc signed (built with codesign --sign -) but not notarised with Apple. On some macOS versions, Gatekeeper may still show a “cannot be opened because it is from an unidentified developer” dialog on the very first run.

    If that happens: right-click (or Control-click) the binary in Finder → Open. This prompts a different dialog that includes an “Open” button. After that first approval the binary runs normally. Alternatively, re-run the xattr -d command above.

  4. 4

    Move the binary to Bun’s bin directory.

    shell 
    mkdir -p ~/.bun/bin
mv ~/Downloads/iris-darwin-arm64 ~/.bun/bin/iris
    ⓘ PATH is already set by Bun

    If you installed Bun, ~/.bun/bin is already on your $PATH — no extra configuration needed. If iris is not found after moving the binary, add this fallback line to ~/.zshrc and reload: 

    export PATH="$HOME/.bun/bin:$PATH"
    source ~/.zshrc
  5. 5

    Launch Iris.

    shell 
    iris
# Launches the full-screen TUI. Press q or Ctrl-C to quit.

    On first run, Iris creates its data directories automatically — no manual setup needed.

ⓘ Where your data lives

Iris writes all user data outside the binary, so it persists across updates:

Path Contains
~/.iris/ Config (config.json), field mappings, validation history, submission metadata
~/Documents/Iris/ Generated ILR XML files (Submissions/), custom schemas (Schemas/)

03 Replace an Old Installation

Upgrading Iris is a simple binary swap. Because all user data lives in ~/.iris/ and ~/Documents/Iris/ — not inside the binary — overwriting iris on PATH is safe. Configuration, field mappings, history, and all generated XML are untouched.

✓ Your data survives the upgrade

Swapping the binary replaces only the executable. ~/.iris/ (config, mappings, history) and ~/Documents/Iris/ (ILR XML output, custom schemas) are never touched by the installer — they are created on first run and left alone on every subsequent install.

  1. 1

    (Optional) Keep a rollback copy of the current binary.

    shell 
    cp ~/.bun/bin/iris ~/.bun/bin/iris.bak
    ✓ Rollback

    If the new version has an issue: mv -f ~/.bun/bin/iris.bak ~/.bun/bin/iris and you are back on the previous version immediately.

  2. 2

    Download the new binary.

    shell 
    gh release download --repo foundersandcoders/iris \
  --pattern 'iris-darwin-arm64' \
  --dir ~/Downloads
  3. 3

    Make executable and remove quarantine.

    shell 
    chmod +x ~/Downloads/iris-darwin-arm64
xattr -d com.apple.quarantine ~/Downloads/iris-darwin-arm64
  4. 4

    Overwrite the existing binary.

    shell 
    mv -f ~/Downloads/iris-darwin-arm64 ~/.bun/bin/iris
  5. 5

    Verify the upgrade.

    shell 
    iris help
# Expected: updated version shown in TUI header on launch

    Launch iris to confirm the TUI starts and your existing configurations and history are intact.
